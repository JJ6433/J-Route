問題の定義

既存の旅行アプリは情報は多いですが、「自分だけの日程」を組んでくれない不便さがあります。

解決策

AIが最適動線を計画し、リアルタイム気象・Googleマップデータを一箇所で統合提供するワンストッププラットフォーム。

😤

既存の旅行準備における不便さ

  • Google Maps、天気アプリ、航空券サイトを何度も往復する手間
  • 個人の好みが反映されず、結局手書きでメモを繰り返す不便さ
  • 最適動線なしに組まれた非効率な旅行日程
🤖

J-Routeによる解決策

  • Gemini AIが地域・日付・テーマを分析し、最適化された旅行動線を自動生成
  • Open-Meteoのリアルタイム気象データと天気に基づいた服装ガイドを提供
  • Google Mapsで場所・グルメ・観光地情報を地図と共に連動提供

09 / 26

問題点 · Problem

AI(Gemini) が自然言語 (文章) で回答するため、ビュー (Thymeleaf) でレンダリングが不可能な問題が発生。定型データの抽出ができず、DB 保存も不可.

文章応答 パース失敗 レンダリングエラー

解決策 · Solution

プロンプト内で 「必ず指定された JSON 形式のみで応答すること」 を明示的に強制。さらに 「宿泊先が日程の最初と最後になるよう動線を最適化する」 プロンプトチューニングを適用。

Data Truncation エラー解決 · DB スキーマ マイグレーション

11 / 26

問題点 · Problem

Google Maps の一部 URL が 700文字以上 に達し、DB INSERT 時に Exception 500 · Data Truncation エラーが発生。

既存のカラム型が VARCHAR(255) で設計されており、保存容量が不十分。

解決策 · Solution

エラーログ分析 → DB スキーマ設計の限界把握 → VARCHAR(255) を TEXT 型 にマイグレーション。

コードレベルでの回避ではなく、DB 層からの完璧な原因究明 及び根本解決。

解決プロセス Flow
① 500 エラー確認
→
② サーバーログ分析
→
③ Data Truncation 原因把握
→
④ ERD 再設計
→
⑤ VARCHAR→TEXT マイグレーション
→
⑥ 正常保存 ✓

[Budget] 予算プランナーのUX改善及びバグ修正

23 / 26
関連技術
LocalStorage JavaScript
担当者
イ・スンウク
貢献の要約（問題解決）

複雑な予算データを扱うフロントエンド環境で発生した深刻なデータ消失現象とレンダリングのバグを、原因から分析して完璧に修正しました。

🔥

Problem 1. データの消失

予算をすべて登録した後にリロード（F5）すると、追加したリストがすべて初期化（消失）される致命的なUXの欠陥が発生。

🛠️

Solution 1. LocalStorageの連携

サーバー通信の負荷なしに、ブラウザのlocalStorageに予算状態をリアルタイムで保存・同期し、データ保存の問題を完璧に解決。

🔥

Problem 2. 詳細画像の欠落

予算に含まれた宿泊施設・航空券をクリックして詳細情報ウィンドウを開く際、画像がリンク切れになるか欠落する現象を発見。

🛠️

Solution 2. パラメータのチューニング

openDetailおよびrenderAddedItems関数の呼び出し時に、imageUrlパラメータを明示的に渡すようデータフローを再設計し、エラーを解決。
